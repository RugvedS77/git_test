Contributing to Simple Interest Calculator
First off, thank you for considering contributing! It’s people like you who make open-source projects better for everyone.

## Our Goal
All contributions, bug reports, bug fixes, documentation improvements, enhancements, and ideas are welcome. Whether you are fixing a typo or refactoring the entire math engine, your help is appreciated.

## How Can I Contribute?
### Reporting Bugs
Check the Issue Tracker: Before opening a new issue, please search to see if the bug has already been reported.

Be Specific: Describe the steps to reproduce the bug, the expected behavior, and what actually happened. Include screenshots if applicable.

### Suggesting Enhancements
Open an Issue: Explain the "why" behind your idea. How would this enhancement help other users?

Provide Context: If you have a specific UI/UX idea or a new formula (like Compound Interest), feel free to sketch it out or provide a reference.

### Documentation Improvements
Documentation is just as important as code! If you find a typo, a confusing explanation, or think a tutorial could be clearer, please submit a Pull Request.

## Pull Request Process
Fork the repository and create your branch from main.

Make your changes (ensure your code follows the existing style).

Update the documentation if you are changing any functionality.

Submit the PR with a clear description of what you’ve done and which issue it addresses.

## Community & Conduct
By participating in this project, you agree to maintain a respectful and inclusive environment for everyone.
